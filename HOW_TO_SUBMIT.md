## üöÄ Step-by-Step Submission Instructions

### Step 1: Go to Dev.to
1. Navigate to: https://dev.to/new
2. Make sure you're logged in to your Dev.to account

### Step 2: Copy the Content
1. Open **DEVTO_SUBMISSION.md** in this repository
2. Copy the ENTIRE content (from the frontmatter `---` to the end)
3. Paste it into the Dev.to editor

### Step 3: Configure Post Settings

The frontmatter at the top already includes:
```yaml
---
title: Sanitas Mind - AI-Powered Receipt Health Tracker
published: false
description: A full-stack React + .NET app that helps you make healthier shopping choices using GitHub Copilot AI
tags: githubcopilotclichallenge, react, dotnet, ai
cover_image: https://github.com/user-attachments/assets/9407590b-b809-4532-ba31-0494ac55a8ec
---
```

**Note:** `published: false` means it will save as a draft. Change to `true` when ready to publish.

### Step 4: Add Cover Image
The cover image is already set in the frontmatter:
```
cover_image: https://github.com/user-attachments/assets/9407590b-b809-4532-ba31-0494ac55a8ec
```

This uses the Dashboard screenshot which shows:
- Sanitas Mind branding
- Dark mode interface
- Health Score and KPIs
- Clean, professional design

### Step 5: Preview Your Post
1. Click the "Preview" button in Dev.to
2. Review the formatting
3. Check that all images are displaying correctly
4. Verify the screenshots show properly

### Step 6: Verify Challenge Link
Make sure this line appears at the top:
```markdown
*This is a submission for the [GitHub Copilot CLI Challenge](https://dev.to/challenges/github-2026-01-21)*
```

This is REQUIRED for the post to be counted as a submission!

### Step 7: Publish
1. If everything looks good, change `published: false` to `published: true`
2. Or use the "Publish" button in Dev.to
3. Share the link!
